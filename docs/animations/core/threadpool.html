SGL Thread Pool

Follow a synchronous routine attachment as workers consume an input queue, execute callbacks, fill an output queue, and signal completion.

library/sgl-core/threadpool/threadpool.c
Routine handle
Each attachment owns an input queue, an output queue, callback state, mutex, and condition variable.
Worker sharing
Workers peek the same routine-list entry, then safely dequeue distinct operations.
Synchronous caller
attach_routine() returns only after all operations complete and the handle is removable.
